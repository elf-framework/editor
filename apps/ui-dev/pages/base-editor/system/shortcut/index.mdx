# Shortcut

Shortcut is a simple, yet powerful, tool for creating keyboard shortcuts in your app.

## Regist

Shortcut is a singleton class. You can register a shortcut by calling `editorContext.registerShortcut` method.

```jsx
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e",
  name: "My Shortcut",
  description: "This is my shortcut",
  command: () => {
    console.log("My shortcut is called");
  },
});
```

## Container

`document` is the default container. You can also specify a container by passing a DOM element to the `container` property.

```jsx
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e",
  name: "My Shortcut",
  description: "This is my shortcut",
  container: document.querySelector(".my-container"), // coming soon
  command: () => {
    console.log("My shortcut is called");
  },
});
```

## Options

Shortcut has some options that you can use to customize the behavior of the shortcuts.

### key

The `key` property is required. It is the key combination that will trigger the shortcut.

`key` can be a string.

```jsx
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e",
  name: "My Shortcut",
  description: "This is my shortcut",
  command: () => {
    console.log("My shortcut is called");
  },
});
```

### name

The `name` property is required. It is the name of the shortcut.

### description

The `description` property is required. It is the description of the shortcut.

### command

The `command` property is required. It is the function that will be called when the shortcut is triggered.

command can be a function or a command string.

```jsx
// command is a string
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e",
  name: "My Shortcut",
  description: "This is my shortcut",
  command: "my-command",
});

// command is a function
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e",
  name: "My Shortcut",
  description: "This is my shortcut",
  command: () => {
    console.log("My shortcut is called");
  },
});
```

### container

The `container` property is optional. It is the container that will listen for the shortcut. If not specified, the `document` will be used.

### priority

The `priority` property is optional. It is the priority of the shortcut. The default value is `10`. The higher the priority, the sooner the shortcut will be triggered.

### preventDefault

The `preventDefault` property is optional. It is a boolean that specifies whether the default behavior of the event should be prevented. The default value is `false`.

### stopPropagation

The `stopPropagation` property is optional. It is a boolean that specifies whether the event should stop propagating. The default value is `false`.

### isGlobal

The `isGlobal` property is optional. It is a boolean that specifies whether the shortcut should be global. The default value is `false`.

### isDisabled

The `isDisabled` property is optional. It is a boolean that specifies whether the shortcut should be disabled. The default value is `false`.

### when

The `when` property is optional. It is a function that specifies when the shortcut should be triggered.
The default value is `() => true`.

### mac, win, linux

The `mac`, `win`, `linux` properties are optional.
They are other key string that specify be triggered on the specified platform.

```js
editorContext.registerShortcut({
  key: "ctrl+shift+alt+e", // Default os trigger - mac
  win: "ctrl+shift+alt+e", // It is used windows trigger instead of default os trigger
  name: "My Shortcut",
  description: "This is my shortcut",

  command: () => {
    console.log("My shortcut is called");
  },
});
```

## Key string

KeyString is a string that represents a key combination. It is used to specify the `key` property of a shortcut.

alt, shift, ctrl, meta, and event key code are supported.

```jsx
"alt + shift + ctrl + meta + e";
```

This key is fire when the user press `alt + shift + ctrl + meta + e` keys at the same time.

## Event key code

You can use the event key code to specify the key.

```jsx
"alt + shift + ctrl + meta + 69";
```

## Special keys

You can use the following special keys to specify the key.

```jsx
"alt + shift + ctrl + meta + enter";
```
